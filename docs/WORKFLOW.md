# Учебный workflow

Каждое задание проходит один и тот же цикл:

1. Прочитать урок (lessons/)
2. Написать решение в kata/
3. Запустить:

   make check

4. Если тесты не проходят — исправить код
5. Когда всё прошло:
   - git commit
   - make progress

## Правило

Нельзя считать задание выполненным,
если make check не проходит.

## Минимальный ежедневный ритуал

20–40 минут:

- 1 задание
- 1 тест
- 1 коммит

## Принцип

Лучше маленький законченный шаг,
чем большой незаконченный.
